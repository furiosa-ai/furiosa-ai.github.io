furiosa.torch.lla.tensor_to_sram(tensor: Tensor)#

Copies a tensor to SRAM on an RNGD device.

RNGD’s SRAM is partitioned per Processing Element (PE), and each PE’s SRAM is further divided into slices. Currently, an RNGD PE has num_slices_per_pe (64) slices. If two PEs are fused, the effective number of slices becomes 128.

To be copied to SRAM, the product of the first ‘N’ dimensions of the tensor must equal the total number of target slices (num_slices_per_pe * fusion_factor). Additionally, the length of each slice (i.e., the product of the remaining dimensions) must be a multiple of SRAM_DMA_UNIT_BYTES (256 bytes).

Example

import torch
from furiosa import native_torch
from furiosa.native_torch import lla
from furiosa.native_torch.ir import LoweredShape
from furiosa.torch.lla import *

x = torch.rand([1024 * 1024], device="rngd:0", dtype=torch.bfloat16)
# The examples below assume a fusion factor of 1, so total slices = 64.

# x_sram = tensor_to_sram(x.view([1024, 1024]))  # INVALID: 1024 != 64 (slices mismatch)
x_sram = tensor_to_sram(x.view([64, 16 * 1024])) # VALID: 64 == 64 (slices match), 16*1024 * 2 bytes is multiple of 256
x_sram = tensor_to_sram(x.view([8, 8, 16 * 1024])) # VALID: 8 * 8 == 64 (slices match), 16*1024 * 2 bytes is multiple of 256

y = torch.rand([64, 128], device="rngd:0", dtype=torch.bfloat16)
y_sram = tensor_to_sram(y) # VALID: 128 * 2 bytes = 256 bytes, which is a multiple of 256
y = torch.rand([64, 130], device="rngd:0", dtype=torch.bfloat16)
# y_sram = tensor_to_sram(y) # INVALID: 130 * 2 bytes = 260 bytes, which is NOT a multiple of 256
Parameters:

tensor (torch.Tensor) – The tensor to copy. Its shape must conform to the SRAM slicing rules.

Returns:

torch.Tensor – The tensor copied to RNGD’s SRAM.
