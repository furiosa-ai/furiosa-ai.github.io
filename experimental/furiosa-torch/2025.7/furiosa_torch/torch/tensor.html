torch.Tensor in RNGD

torch.Tensor in RNGD#

Furiosa-Torch provides seamless integration with PyTorch to execute tensor operations on RNGD, FuriosaAI’s 2nd-generation NPU. This section introduces how to move tensors to the device, execute operations, and utilize both DRAM and SRAM memory spaces within the NPU.

You can move tensors to the RNGD device by using the standard torch.Tensor.to() method with a torch.device specifying rngd:{index}.

For more advanced use cases, you can copy tensors into SRAM, which provides faster access for certain operations. However, due to architectural constraints, you must first move the tensor to DRAM before copying to SRAM.

To match the internal hardware layout (64 slices), the high-rank shape of the tensor must match the number of slices. For example: Tensor.reshape([64, 4, 256]) or Tensor.reshape([32, 2, 4, 256]), where 32 * 2 = 64 slices.

import torch
from furiosa.torch.lla import tensor_to_sram

# Copy to RNGD dram, `.to` triggers DRAM allocation and copy from CPU to RNGD.
# To specify device, you can use "rngd:0", range is available from 0 to 7.
dram_device = torch.device("rngd:0")

t0_cpu = torch.rand([256, 256], dtype=torch.bfloat16)
t0_rngd = t0_cpu.to(dram_device)
t1_cpu = torch.rand([256, 256], dtype=torch.bfloat16)
t1_rngd = t1_cpu.to(dram_device)
# Run on RNGD, torch RNGD backend 1) obtains target Aten operator, 2) convert into
# `Graph` and compile to `EDF` which is RNGD runnable format, and
# 3) run on RNGD using `HAL` interfaces.
# It fallbacks to CPU when compiler failed to compile.
assert torch.all((t0_rngd + t1_rngd).cpu() == t0_cpu + t1_cpu)

# Use of SRAM examples as follows.
# To copy data to SRAM, must copy data to DRAM first.
tensor = torch.rand([256, 256]).to(dram_device)
# RNGD PE consist of 64 slices, and each slice has 64 slices. (See the hardware modeling document for more details)
# To copy data SRAM, high rank shape must be matched with num slicese.
tensor_sram = tensor_to_sram(tensor.reshape([64, 4, 256]))
# Or, can split into multiple shapes for axes for slices, 32 * 2 == 64
tensor_sram = tensor_to_sram(tensor.reshape([32, 2, 4, 256]))

# In case of SRAM, we don't support eager execution.
# Place tensor to DRAM when you need to execute torch code eagerly.
# For example, the following code will fail.
try:
    tensor_sram + tensor_to_sram(torch.rand([32, 2, 4, 256]).to(dram_device))
except Exception as e:
    print(e)
