Custom Operator Extension

Custom Operator Extension#

This document introduces how to implement a custom operator extension for execution on FuriosaAI’s RNGD (2nd-generation NPU). While Furiosa’s PyTorch integration continues to support a wider range of operators natively on RNGD, there are cases where certain operations may not yet be supported or may require custom implementations to optimize performance or functionality.

To address this, Furiosa provides the furiosa.torch.extension module, which enables users to define and register custom implementations of PyTorch operators. These implementations can prioritize execution on RNGD and gracefully fall back to CPU when necessary.

The following example demonstrates how to define a custom implementation of the torch.mm operator using a function called custom_mm_out.

"""
Implement custom extension for RNGD.
"""

import torch
from furiosa.torch.cache import run_graph
from furiosa.torch.extension import move_to_rngd, move_to_cpu
from furiosa.native_torch import UnsupportedOpError


def custom_mm_out(*inputs, out=None, **kwargs):
    print("custom_mm_out called")
    if len(inputs) >= 2:
        rngd_tensor_args = move_to_rngd(*inputs)
    else:
        rngd_tensor_args = inputs

    try:
        run_graph(torch.ops.aten.mm, rngd_tensor_args, [out], **kwargs)
    except UnsupportedOpError as e:
        cpu_tensor_args = move_to_cpu(*inputs, as_tuple=True)
        cpu_out = torch.ops.aten.mm(*cpu_tensor_args, **kwargs)
        out.copy_(cpu_out)


"""
Currently, torch does not support 'overriding' already registered ops.
To show the custom extension, make it comment out.
It will work for the unimplemented ops.
"""
# from furiosa.torch.extension import rngd_lib
# rngd_lib.impl("mm.out", custom_mm_out, "PrivateUse1")

# Executes the custom mm
device = torch.device("rngd:0")
x_cpu = torch.randn(256, 256, dtype=torch.bfloat16)
y_cpu = torch.randn(256, 256, dtype=torch.bfloat16)
x = x_cpu.to(device)
y = y_cpu.to(device)
answer = torch.mm(x_cpu, y_cpu)
z = torch.mm(x, y)
torch.allclose(z.cpu(), answer, atol=0.02, rtol=0.02)
